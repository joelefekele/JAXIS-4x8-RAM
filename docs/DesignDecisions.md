# Design Decisions

## Introduction

This document explains the engineering decisions made during the design of the JAXIS 4×8 RAM. Every major subsystem was selected to satisfy a specific design requirement while maintaining simplicity, modularity, and expandability.

---

# Why Build RAM from Discrete Logic?

Instead of using a commercial RAM integrated circuit, the project implements memory using standard 74HC logic devices.

This approach makes every stage of memory operation visible and understandable, allowing the behavior of registers, buses, decoders, and control logic to be observed directly.

---

# Why a 4×8 RAM?

A 4-word × 8-bit memory was selected because it is large enough to demonstrate all fundamental RAM operations while remaining practical to prototype on breadboards.

This architecture also provides a solid foundation for future expansion.

---

# Why the 74HC574?

The 74HC574 was selected because:

- It stores eight bits in one integrated circuit.
- All eight bits are written simultaneously.
- It provides tri-state outputs through the Output Enable (OE) input.
- It accurately represents a single byte-wide memory register.

Although the device does not include a dedicated Write Enable pin, external logic was developed to generate controlled write operations.

---

# Why Use a 74HC139 Address Decoder?

The RAM contains four memory registers.

A two-bit address bus provides four possible addresses.

The 74HC139 converts the binary address into one active register selection signal.

This simplifies register selection while allowing the design to scale to larger memory systems.

---

# Why Implement External Write Logic?

Because the 74HC574 has no Write Enable input, the write function is implemented externally.

Each register receives its own gated clock generated by a three-input AND gate.

Each AND gate combines:

- Clock
- Write Enable (WE)
- Decoder Output

A register stores data only when all three conditions are satisfied.

This guarantees that only the selected register updates its contents.

---

# Why Use a Shared Data Bus?

Rather than connecting every register directly to every device, the RAM uses one shared 8-bit data bus.

Advantages include:

- Reduced wiring complexity
- Modular hardware design
- Easier debugging
- Simplified expansion
- Architecture similar to real computer systems

The shared bus becomes the central communication path of the RAM.

---

# Why Use the 74HC125?

The data input switches and memory registers both communicate through the same data bus.

Without isolation, both devices could attempt to drive the bus simultaneously.

The 74HC125 solves this problem by disconnecting the DIP switches during read operations.

This prevents bus contention while allowing both write and read operations to use the same shared bus.

---

# Why Manual Controls?

The first version of the RAM uses:

- DIP switches
- Push buttons
- LEDs

instead of a microcontroller.

This design allows every signal to be manually controlled and observed.

Manual operation also simplifies debugging and demonstrates the complete hardware operation of the memory system.

---

# Why Modular Design?

Each subsystem performs one dedicated function.

Examples include:

- Address selection
- Address decoding
- Data storage
- Write control
- Read control
- Bus buffering

This modular approach simplifies troubleshooting and allows individual subsystems to be upgraded without redesigning the entire RAM.

---

# Expandability

The architecture has been designed for future expansion.

Possible improvements include:

- Larger memory capacity
- Automatic read logic
- PCB implementation
- CPU interface
- ALU integration
- Complete computer architecture

The current RAM module serves as the memory subsystem for future JAXIS hardware projects.

---

# Conclusion

Every major design decision in the JAXIS 4×8 RAM was made to balance simplicity, educational value, modularity, and scalability.

The resulting architecture provides a clear demonstration of how digital memory operates while establishing a reusable foundation for more advanced computer engineering projects.
