# Hardware Architecture

## Overview

The JAXIS 4×8 RAM is a hardware-only memory system designed entirely using discrete 74HC-series logic integrated circuits.

Unlike RAM implemented with programmable devices or microcontrollers, the JAXIS RAM demonstrates how computer memory can be designed from first principles using fundamental digital logic components.

The architecture separates memory into independent hardware modules that communicate through a shared 8-bit data bus. Each module performs one dedicated function, resulting in a modular design that is easy to understand, debug, test, and expand.

---

# Design Objectives

The JAXIS RAM was designed to:

- Understand RAM architecture from first principles.
- Implement computer memory using only discrete logic ICs.
- Demonstrate hardware design without programmable devices.
- Simplify debugging through modular hardware.
- Create a scalable architecture for future computer development.

---

# System Architecture

The JAXIS RAM is divided into the following hardware modules.

- Address Input
- Address Decoder
- Write Logic
- Register Bank
- Shared Data Bus
- Data Input Buffer
- Manual Read Logic
- LED Display
- Power Distribution

Each module performs one specific task.

---

# Address Input

The Address Input module consists of a 2-bit DIP switch.

The Address DIP Switches generate the binary address used during WRITE operations.

Outputs:

- A1
- A0

These signals connect directly to the 74HC139 Address Decoder.

---

# Address Decoder

The Address Decoder is implemented using one 74HC139.

Its only responsibility is to decode the binary address into one register select signal.

Outputs:

- Y0
- Y1
- Y2
- Y3

The decoder is used only during WRITE operations.

It does not control READ operations.

---

# JAXIS Write Logic

The JAXIS RAM implements external write logic because the 74HC574 does not include a dedicated Write Enable input.

Each register receives its own independent clock signal generated by a three-input AND gate.

Each AND gate combines:

- Write Enable (WE)
- Decoder Output (Yn)
- Clock Push Button (CLK)

Only when all three signals are active does the selected register receive a clock pulse.

This guarantees that:

- Every register receives the shared data bus.
- Only the selected register stores the data.
- All non-selected registers preserve their previous contents.

---

# Register Bank

The register bank consists of four identical 74HC574 octal D-type registers.

Each register stores one 8-bit word.

Each register receives:

- Shared Data Bus
- Individual Clock Input
- Individual Output Enable

# Shared Data Bus

The shared data bus is the communication backbone of the JAXIS RAM.

It connects:

- Data DIP Switches
- Two 74HC125 Buffers
- Register Inputs
- Register Outputs
- LED Display

The shared data bus never stores information.

Its only responsibility is transporting data.

---

# Data Input Buffer

The Data Input Buffer consists of two 74HC125 Quad Tri-State Buffers.

Together they buffer all eight data lines.

### WRITE Mode

The buffers are enabled.

The Data DIP Switches drive the shared data bus.

### READ Mode

The buffers are disabled.

The Data DIP Switches enter High Impedance and disconnect from the shared data bus.

This prevents interference with data supplied by the selected register.

---

# JAXIS Manual Read Logic

Unlike many RAM designs, Version 1 of the JAXIS RAM does not use automatic read selection.

Instead, each register has its own manual Output Enable switch.

Each switch directly controls the OE input of one register.

Advantages of this approach include:

- Simplified prototype design.
- Easier debugging.
- Independent testing of every register.
- Clear observation of register behavior.

Only one register should have OE LOW during a READ operation.

All remaining registers remain in High Impedance.

---

# LED Display

The LED display monitors the shared data bus.

During WRITE operations, the LEDs display the byte selected by the Data DIP Switches.

During READ operations, the LEDs display the byte stored inside the selected register.

---

# Modular Design Philosophy

Every hardware module in the JAXIS RAM performs one dedicated responsibility.

| Module | Responsibility |
|---------|----------------|
| Address Input | Generate memory address |
| Address Decoder | Select write destination |
| Write Logic | Generate register clock |
| Register Bank | Store data |
| Shared Data Bus | Transport data |
| 74HC125 Buffers | Isolate Data DIP Switches |
| Manual Read Logic | Select register output |
| LED Display | Display bus data |

Separating these responsibilities simplifies testing, troubleshooting, documentation, and future expansion.

---

# Future Development

The architecture was intentionally designed to support future improvements without redesigning the memory subsystem.

Planned future enhancements include:

- Automatic Output Enable logic.
- Larger memory arrays.
- Expanded address bus.
- CPU interface.
- ALU integration.
- Complete discrete logic computer.

---

# Summary

The JAXIS 4×8 RAM demonstrates how computer memory can be implemented using discrete digital logic integrated circuits.

By combining a shared data bus, external write logic, manual read control, tri-state buffering, and modular register design, the architecture provides a clear and scalable foundation for learning computer engineering while serving as the memory subsystem for future JAXIS computer projects.
